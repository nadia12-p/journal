Creating a User-Based Recommender in 5 minutes

I learned how the Mahout Recommender works, I was able to use this for my Movie Recommender for this week’s assignment. However, what I think was most important from this was the learning it gave me to figure out which data is important for a specific method and that we may have to preprocess the data before we use any method.

Test Driven Development

I learned that this methodology helps us to not code blindly, having tests from the beginning lets us catch mistakes and see if the approach we’re taking is the correct one or if we should change it.

Unit Test

With unit test we check if a small part of the code works, this will be helpful to implement because we can see if the issue is only at a specific point or if we need to change our whole approach.

Xunit

I learned that depending on the language that we use the xUnit is different. For example, this week I used JUnit which allowed me to test my program in java. This was useful because I was able to test my program and see that it was actually working correctly.

Mocks Aren't Stubs

I learned that there are different philosophies behind the Test-Driven Development: Classical TDD and Mockist TDD. I think I still need more practice to work out which one I fall under but it will be important to understand them so I can apply the one I’m most comfortable with.

Java Build Tools: How Dependency Management Works with Maven, Gradle and Ant + Ivy

This was useful because I was able to define dependencies on maven that helped me out with the problem of the week. I now know where dependencies should be added and this way I know if there’s one missing from my project.

Missing semester: Editors (Vim)

I learned more about Vim, which can help me out in the future to read text files and programs from the console. I think this will be useful because there may be servers without a graphic interface.

Missing semester: Version Control (Git)

I learned more commands from Git. I think the one that was really useful for me this week was git commit because it let me store the changes in my code and I could look back on it if I did something wrong when I used git log.

Pale Blue Dot

The future of programming

I think that only limiting yourself to one type of thinking is something very dangerous, this is something that can be hard to break out however because if it works, we don’t want to change it. Keeping a mindset that you don’t know what you’re doing is something that I want to do so I can be more open to new ideas that may be better than the ones I'm currently using.

Inventing on principle

What I learned was that experiences help us analyze ourselves. It’s important to try out different skills and ideas so we know if they’re useful, need improvement, or aren’t important at the moment. Making a principle is not something quick but different experiences may guide us to discover it.

Machine learning - google

Making patterns is a key way for our brains to learn new things. Patter matching can improve the rate at which we learn immensely. Making a computer do the same with only input data can be very useful from a business perspective because it can show the users certain products that they’ll be more likely to buy. Also, with the pattern matching the computer may be able to identify dangerous paths but because the computer learned a similar solution to a similar problem at its training stage, now it knows what to do.

I want to get better at accepting failure and to envision each failure as a step closer towards making something better.

Quantum physics and quantum computers

Through all the material that related to quantum physics I was able to understand a bit more about how it is that it works. The double slit experiment helped me to comprehend the behavior of quantum particles and it was thanks to this that I had a clearer image of QUBITS. I think a lot of stuff went over my head and I definitely need to read more about the topic but I found it interesting that this field may open the doors for new computers so we humans aren’t physically restricted to new inventions. The leap to quantum computers may take a while though, but I see it as when we had big computers that took up a lot of space. Maybe in some years there’ll be a breakthrough and quantum computers may be the common thing to use. I liked this topic and I think it’s useful to learn more about it so we know how our devices work and have an idea of what we can ask of them and what limits they have, all this with the idea to keep improving.
