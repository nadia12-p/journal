What I learned: June 28 - July 4, 2022

Last Demo

This Tuesday we had our last demo, and there was a lot of learnings from that meeting. I realized we were missing some context on some things, and it’s best to be clear about the information we present because we can make the client, or anyone that hasn’t been working with the team directly, confused and worried about the state of the project. I also learned that it’s best to use a simple language that anyone can understand when we’re doing demos or presentations so that anyone can follow the progress being shared. And that if we’re going to say something technical to be prepared to explain it in a clear and easy way, and that it’s best to not put things we aren’t 100% sure about.

Open Source Projects

This week I was assigned to two issues from two open-source projects. The first one I’ve contributed before, the feature requested was to redesign the pause menu of the game, you can look at the issue here. I’ve researched the FXGLMenu class to learn how to customize its methods for my project’s needs. I’ve made some progress that can be found here.

I'm waiting for a response from the maintainers to know what final layout they want for this pause menu, meanwhile I've added three buttons (back, controls, quit game). However, by clicking on the buttons with the mouse, the functionality works fine but the button doesn’t look like something is hovering over it. I have to wait for a response to know all the buttons that wish to be added, meanwhile I'm working on the functionality issues mentioned before.

The other project I’ve been assigned to can be found here. The version flag doesn’t work, and the help flag is an enhancement. I'm researching how to do command line interfaces on typescript, I'm hoping that with the experience from the ledger project I can solve this issue quickly, as the pause menu issue from the first open-source project is taking me more time than expected.
