What I learned: May 24-30, 2022

Open Source Project

This week I worked on another issue in the dinosaur exploder project. On this issue, I added a feature, whenever the spaceship receives damage the game screen will flash red for half a second to indicate damage. I learned a lot from this issue. The most important thing I believe I learned was to select the important parts of example code and adapt it to my own project. I was researching various libraries of java FX that I thought were going to be useful like Animation, ScaleTransition, SequentialTransition, etc. At the end, I didn’t use them because my issue could be resolved in a simpler manner.

I learned more about lambda functions. I was having an issue where the parameters weren’t compatible, I realized the issue was that I was giving it a null value instead of a SequentialTransition. I didn’t use this function in my final answer but it helped me to understand a bit more of the syntaxis of lambda functions.

I learned that the easiest way to affect the entire background for my project was to use the getGameScene() method and attach the red background to it. However, it took me a while to understand this because the examples I was seeing all created an animation first and ran it in an overlapped view of the screen.

I tried following the examples but the entity to where I was attaching the animation was already being modified by another function in the program, and it’s why when I called the method runOnce() it wasn’t respecting the duration of which I assigned or when I was expecting the action to run. To solve this, I used the getGameScene() method and it changed the whole background and did it when it was expected.

It took me a while to understand how the examples were using this method because I didn’t saw it declare anywhere and thought that I had to declare it in my project for it to work. But then I saw that with one of the libraries of the project I was able to call it and use it. Understanding where it came from and if I could use it in my project without declaring too many things was the biggest problem I had because the example I was looking at used a lot of functions and libraries that weren’t necessary in my project, and understanding each one to see if they could help me solve my issue was what took most of my time this week.

This was good because I learned more about java, java fx and fxgl, but it took a lot of time to solve my issue so I think next time I'm looking at an example I should be more selective on what I spend my time researching about, and to really think if it could be of use for my project.
