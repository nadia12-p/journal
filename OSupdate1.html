What I learned: May 17-23, 2022

Open Source Project

This week I learned how to look for open-source projects on GitHub and select one I could work on. I picked an issue after a couple of days that seemed to be a good starting point for myself, as I’ve never worked on someone else’s project before. I learned about JavaFX, especially hitboxes, this week. Because the issue I picked was to fix the hitbox size of an entity in the project. At the end of the day, I was only asked to edit the picture and fix the size of it. While it wasn’t very technically challenging it did help to learn how to work on a branch of a project, make significant commits, make a pull request and communicate with the maintainers of the project. Also, this issue helped me to read other people’s code, it also gave me an idea of how I should structure my code when working on bigger projects that will be shared and extended by other people. I picked another issue after this one was done, that seems more challenging, I’ll be working on this one the following week. I have an idea of the question I’ll ask and answer regarding this project but I want to keep researching JavaFX before I structure it.

How do CSS Preprocessors work?

On the other hand, I’ve been researching more about the question I picked: How do CSS preprocessors work? I’m still researching more about it but I got the general idea that preprocessors will extend the default capabilities of CSS. To understand more about it, I downloaded the SASS preprocessor with NodeJS and I’ve been researching how the translation of a language works when you run the script. I learned that preprocessors give you more freedom when programming, they give you more options that regular CSS doesn’t like variables and functions. I had a bit of trouble understanding how to convert the Sass file to a regular CSS file with npm but now that I know how to do that, I’ll be focusing on answering how this conversion is actually possible to answer my question.
